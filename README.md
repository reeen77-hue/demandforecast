###1. プロジェクト概要（Why）###

本プロジェクトは、アパレル小売における日次の販売実績データをもとに、
商品別・属性別（性別・年齢）の需要を予測し、
在庫配分・売場設計・販売計画の精度向上を目的として作成した。

現場では、

「どの商品が、どの層に、いつ売れるのか」

「感覚ではなく、数値で判断できるか」

が重要である一方、
Excel集計や経験則に依存した意思決定になりやすい課題がある。

そこで本プロジェクトでは、
現場で実際に取得・管理可能な粒度のデータを用いて、
Pythonによる需要予測の基本的な流れを再現した。

###2. データ設計（現場視点）###
使用データ構成

本プロジェクトでは、以下の列を持つCSVデータを使用している。

date：販売日

product_code：商品コード

gender：性別

age_group：年齢層

sales_qty：販売数量

この設計にした理由

日次データ
→ 曜日・週末・イベント影響を将来的に取り込める

商品コード単位
→ SKU別の在庫・売場判断に直結する

性別・年齢層
→

売場の打ち出し変更

フェイス配分

VMD・販促判断
に活用可能

実務では「商品別合計」だけでなく、
「誰に売れているか」まで分かることで初めて意味のある予測になるため、
本設計を採用した。

3. 手法とモデル比較
実施した内容

日付データから以下の疑似特徴量を作成

曜日

月

週末フラグ

学習モデル

線形回帰（Linear Regression）

ランダムフォレスト（Random Forest）

比較した理由

線形回帰は

シンプルで解釈性が高い
一方で、

非線形な関係を捉えにくい

ランダムフォレストは

複雑なパターンを捉えられる
一方で、

モデルの解釈が難しい

**「精度」だけでなく、「使い分けの考え方」**を理解するため、
両モデルを比較した。

4. 結果

線形回帰

MAE（平均絶対誤差）：約 ◯◯

ランダムフォレスト

MAE（平均絶対誤差）：約 ◯◯

ランダムフォレストの方が誤差は小さく、
非線形な需要変動をより捉えられていることが確認できた。

また、

散布図

残差プロット

を用いて、
どの程度予測がズレているかを可視化した。

5. 考察・課題

データ量が少ないため、
季節性やイベント影響は十分に反映できていない

実務では

在庫数

入荷予定

天候

価格・値引き
などの要素が需要に影響する

本プロジェクトは、
**「需要予測の流れを理解するための最小構成」**として位置付けている。
